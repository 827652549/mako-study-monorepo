练习 4 · 异步闭包 stale state 问题

问题根因
React 的 useState 每次 render 产生新的 state 快照。
当你在 异步回调（setTimeout / SSE onmessage / Promise.then） 中读取 state，
读到的是 回调被创建时 的那个快照（已过期的旧值），而不是最新值。

这个现象叫 stale closure（陈旧闭包）。

在 SSE 场景中：
  sendMessage() 创建了 onmessage 回调 → 闭包捕获了当时的 messageList（比如 []）
  500ms 后第一帧来了 → onmessage 读到的 messageList 还是 []
  300ms 后第二帧来了 → onmessage 读到的 messageList 仍然是 []（!）
  结果：每帧都在空列表上追加，只保留最后一帧
useLatestState 原理（项目里的解法）
// ─── 项目源码 ───
export const useLatestState = (initialValue) => {
  const [state, setState] = useState(initialValue);
  const stateRef = useRef(initialValue);        // ref 不受 render 影响

  const updateState = (value) => {
    stateRef.current = value;   // ← 同步更新 ref（永远是最新值）
    setState(value);            // ← 触发 re-render
  };

  return [state, updateState, stateRef];
};

// ─── 在 SSE onmessage 中使用 ───
const [messageList, setMessageList, messageListRef] = useLatestState([]);

const onmessage = (frame) => {
  // ✅ 读用 ref（永远最新），写用 setState（触发渲染）
  const newList = messageListRef.current.map(item => ...);
  setMessageList(newList);
};
